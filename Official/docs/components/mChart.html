mChart

純 SVG、零依賴、Highcharts 風格 options API 的折線 / 平滑曲線圖元件

適合閱讀對象

適合閱讀對象：UI / 設計、前端 F2E、QA。

角色確認重點
UI / 設計線色 / 補值色（走專案 token）、頭尾留白、補值段虛線、tooltip 樣式
前端 F2Eoptions API（config/xAxis/yAxis/tooltip/series）、補值邏輯、座標 / 刻度演算、互動 tooltip
QA缺月補值與虛線、頭尾標籤跨年繞回、hover 命中區邊界、虛線點不出 tooltip、可點擊 anchor

此元件由 Nuxt 自動匯入，元件名為 BuyMChartMain（對應 components/buy/mChart/Main.vue）。

核心設計：父系只丟原始資料 + config，補缺月 / 算類別軸 / 畫圖全部封裝在元件內。座標、路徑、補值三類可重用邏輯拆在 components/buy/mChart/.composables/（dot 開頭，不被 Nuxt 自動匯入，以相對路徑引入）。

1. 元件介紹 F2E

mChart 是一個不依賴任何第三方圖表庫的折線圖元件，用原生 SVG 繪製。設計動機是避免引入 ECharts / Highcharts 這類「全包式黑盒」帶來的維運與客製成本 —— 像「補值段落要畫虛線」這種需求， 自己刻反而最單純。API 命名刻意對齊 Highcharts（config / xAxis / yAxis / tooltip / series），降低學習成本。

  • 線型：line（直線）/ spline（Catmull-Rom 平滑曲線），逐段渲染。
  • 補值：成交均價趨勢常缺月，元件可依設定補上缺月並以虛線標示推估段落。
  • 零依賴：座標轉換為國中數學的線性內插，全部手刻；顏色走專案 CSS token（--green-8b0d 等）。
  • RWD：以 ResizeObserver 量測寬（.m-chart）與高（.m-chart-container），resize 自動重算，無固定 height 設定。

2. 檔案架構 F2E

檔案職責
mChart/Main.vue主程式：合併 options 預設、量測寬度、組幾何、渲染 SVG、處理 hover / tooltip
.composables/useChartScale.js座標轉換：linear（線性 scale）、point（等距 scale）、niceTicks（漂亮刻度）
.composables/useChartPath.js逐段路徑：lineSegments / splineSegments / buildSegments，每段帶 dashed
.composables/useChartFill.js補值：fillMonths（缺月補值）、monthShift（前後月推算，給頭尾標籤用）
職責分層：父系丟原始 priceTrend + config → Main.vue 依 config.fillGap 呼叫 fillMonths 補缺月、產生類別軸、畫圖 → useChartFill 是純補值演算法。圖表核心不認識「月份」這種特定領域邏輯，月份只是 padFormatter / fillMonths 的預設策略。

3. Props 總覽 F2E

五個 prop，皆為 options 物件 / 陣列。xAxis / yAxis / tooltip 既可當獨立 prop，也可內嵌在 config 裡；同時存在時 獨立 prop 優先（覆蓋 config 內的同名設定）。

Prop型別說明
configObject圖表主設定（線型、標題、尺寸、補值、留白…），可內嵌 xAxis/yAxis/tooltip
xAxisObjectx 軸：類別標籤、頭尾留白、邊界標籤
yAxisObjecty 軸：值域、刻度、單位、格式
tooltipObjecthover 提示：啟用、虛線點顯示、互動、格式
seriesArray資料系列 [{ name, color, data }]
setClassObject各部位 class { main, container, title, xAxis, yAxis }

setClass 對應

key套用部位備註
main最外層容器 .m-chart一般 HTML；可設整體高度來源
containerSVG .m-chart-container圖高來源：在此設高度（如 h-full），元件會量測它
title標題 <h3>一般 HTML，text-* 即可改色
xAxisx 軸刻度文字SVG <text>，改色用 fill-*（非 text-*）
yAxisy 軸刻度文字（含首尾值）SVG <text>，改色用 fill-*

class 附加在預設樣式之後；SVG 文字顏色由 fill 控制，故改色要用 fill-[--xxx] 而非 text-[--xxx]。

尺寸：寬度量測自最外層 .m-chart、高度量測自 .m-chart-container（SVG），皆由 ResizeObserver 在 resize 時自動重算，已無 config.height。要控制圖高，用 setClass.container（如 h-full）或外部 CSS 對 .m-chart-container 設高度；未設時用後備值 260px。

padding × padLabel 組合

paddingpadLabel效果
false—折線貼齊左右邊緣，無頭尾標籤

7. tooltip 詳解 F2E QA

欄位型別預設說明
enabledbooleantrue是否啟用 hover 提示
showDashedbooleanfalse補值 / 推估（虛線）點是否顯示 tooltip
interactivebooleanfalse可滑入 tooltip 點擊內容（如 anchor）；滑出 band 會延遲 200ms 關閉
formatter(ctx)=>stringnull自訂提示 HTML（走 v-html）

formatter 收到的 ctx：

{
  index,            // 類別索引
  category,         // 該索引的 x 標籤
  points: [         // 各 series 在此索引的點
    { series, color, value, dashed, raw }   // raw＝原始資料點（date/yearMonth…）
  ]
}

預設輸出 <b>{category}</b> 後接每條 {name}：{value} {unit}，推估點補上「（推估）」。要顯示其他欄位（如單位「/坪」或日期），用 formatter 搭配 raw。

interactive:true 時 tooltip 改 pointer-events-auto 才能點 anchor。但 v-html 內的 <a href> 是原生連結（點了整頁導向）；若要 SPA 路由（NuxtLink / router.push 不重整），需在 tooltip 容器上做 click 事件代理，v-html 無法直接渲染 <NuxtLink>。

8. series 與資料格式 F2E

每條 series：{ name, color?, data }。資料點統一契約只有 label（x 類別）與 value（y 值）；其餘欄位原樣保留，tooltip 由 ctx.points[].raw 取用。

data: [
  { label: '11月', value: 88.9, date: '2023-11-01', yearMonth: '112/11' },
  { label: '1月',  value: 62.3, date: '2024-01-01', yearMonth: '113/01' },
]
  • 圖表只認 label / value；要顯示其他值（日期、單位…）在 tooltip.formatter 透過 raw 取得。
  • 補缺月：搭配 config.fillGap 時需要 date 欄位才能算間距；補出的月份標 dashed:true（畫虛線）。
  • 後備：仍容忍舊欄位 monthLabel→label、unitPrice→value，但建議統一用 label/value。
虛線（dashed）語意：折線逐段渲染，某段兩端只要任一端 dashed，該段就畫成虛線（mutedColor + stroke-dasharray），端點也以 mutedColor 標示 —— 用來表達「這是補值 / 推估，非真實成交」。

9. 補值邏輯：useChartFill F2E

fillMonths(raw, { fillGap, value }) 把稀疏月資料整理成連續陣列，回傳 { categories, data }。

  • 缺月偵測：用 date 轉絕對月份索引（年*12+月，避開時區），排序後找出間距。
  • 補值：value:'interpolate' 以前後兩個月線性內插；value:<數字> 一律補該固定值。補出的點 dashed:true。
  • 容錯：值讀 value ?? unitPrice ?? y、標籤讀 label ?? monthLabel ?? name ?? 月份；每點保留 raw 供 tooltip；沒有 date 就只透傳、不補。
  • monthShift(label, delta)：由 '11月' 推算前/後 N 個月（'1月'→'12月'），跨年自動繞回；非 N月 回空字串。作為 xAxis.padFormatter 預設。

由 config.fillGap 決定要不要補：

// Main.vue 內部，每條 series 都跑
const { categories, data } = fillMonths(s.data ?? [], {
  fillGap: cfg.value.fillGap,
  value: cfg.value.fillValue,
})

10. 座標與刻度：useChartScale F2E

  • linear(domain, range)：數值域 → 像素域 的線性映射（y 軸、x 軸都用它）。
  • point(count, range)：等距 scale 工具（通用，目前 x 軸改用 domain + linear）。
  • niceTicks({min,max,count,interval})：算出 0 / 20 / 40 … 這種漂亮刻度，回傳 { min, max, step, ticks }。

x 軸 domain / range（Main.vue）

  • xDomain（index 值域）：有 padLabel 時 [-1, count]（頭尾各留一格）；否則 [0, count-1]；min/max 可覆寫。
  • xRange（像素範圍）：只有 padding 沒 padLabel 時頭尾各內縮 padPx；否則滿版。
  • bandWidth：依 xRange 跨距 / domain 跨距 推算，兩種留白模式 tooltip 都對齊。

11. 互動 / hover 行為 F2E QA

  • 命中區（band）：每個類別一個透明 <rect>，以點為中心展開，但夾在繪圖區 [innerLeft, innerRight] 內，不溢出到 y 軸標籤區或右邊緣。
  • 進 / 出：band mouseenter 設 active、mouseleave 關閉；移到 y 軸區（離開 band）即時關閉。
  • 虛線點不出 tooltip：當前 hover 點若全為虛線（推估），且 tooltip.showDashed=false，則隱藏 tooltip。
  • 互動 tooltip：interactive:true 時滑出 band 延遲 200ms 關閉，游標移進 tooltip 取消關閉，可點內容；移出 tooltip 才關。

12. 使用範例 F2E

社區成交均價趨勢（專案實際用例）

<script setup>
const buyHouse = useBuyHouseStore()
const { community } = storeToRefs(buyHouse)

// 父系只丟原始資料；補缺月由元件內依 fillGap 處理
const priceSeries = computed(() => [
  { name: '成交均價', data: community.value.priceTrend ?? [] },
])
</script>

<template>
  <BuyMChartMain
    :series="priceSeries"
    :config="{
      type: 'line',
      title: '近期成交均價趨勢 (萬/坪)',
      fillGap: true,
      xAxis: { padding: true, padLabel: true },
    }"
  />
</template>

平滑曲線 + 固定間距 40px

:config="{ type: 'spline', xAxis: { padding: 40 } }"

自訂 y 軸單位與刻度

:config="{ yAxis: { unit: '元', tickInterval: 1000 } }"

互動 tooltip + 可點擊連結

:config="{
  tooltip: {
    interactive: true,
    formatter: (ctx) => `${ctx.category}<br><a href="/detail">看詳情</a>`,
  },
}"

非月份 x 軸（自訂邊界標籤策略）

:config="{
  xAxis: {
    categories: ['Q1','Q2','Q3','Q4'],
    padding: true,
    padLabel: true,
    padFormatter: (label, dir) => '',  // 不補字、只留空格
  },
}"

13. 狀態 / 例外行為 QA

  • 無資料：series[].data 為空 → 無類別、無折線，圖面僅有 y 軸刻度，不報錯。
  • 單點：只有一個資料點時置中顯示。
  • 非月份 / 無 date 資料：fillMonths 偵測到缺 date 就不補、直接透傳；padFormatter 預設對非 N月 標籤回空字串。
  • padLabel 但 padding 關：padLabel 被忽略（沒距離就沒空間放標籤）。
  • SSR：首次渲染寬 600 / 高 260 後備值，掛載後由 ResizeObserver 更新為實際量測值。

14. 維護注意 F2E

  • 顏色一律走專案 token（--green-8b0d / --gray-ccce / --gray-e5 / --gray-666），勿寫死色碼。
  • .composables 是 dot 開頭資料夾，不會被 Nuxt 自動匯入；Main.vue 以相對路徑引入，外部要用 useChartFill 可走 @components/buy/mChart/.composables/useChartFill.js。
  • 「補值」是資料層概念（useChartFill），圖表核心不該知道月份等領域邏輯；要支援新的類別軸，給對應的 padFormatter / 在資料層自備補值即可。
  • 若要多 series：y 軸刻度、類別軸取所有 series 的聯集 / 最大長度；tooltip 會列出該索引所有有值的 series。
