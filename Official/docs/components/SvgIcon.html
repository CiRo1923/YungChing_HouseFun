SvgIcon

以 SVG sprite（<use>）渲染專案內建 icon 的共用元件

適合閱讀對象

適合閱讀對象：UI / 設計、前端 F2E、QA。

角色確認重點
UI / 設計可用 icon 清單與命名、尺寸 / 顏色由外層 class 控制（fill-current 繼承 color）
前端 F2Eicon prop 對應的 sprite symbol id、sprite 來源（_svg/*.svg）、dev / prod 路徑差異與快取版號
QAicon 名稱錯字 / 不存在時的破圖行為、HMR 即時更新、RWD 下尺寸是否由 class 正確控制
此元件由 Nuxt 自動匯入，元件名為 CommonSvgIcon（對應 components/common/SvgIcon.vue）。

1. 元件介紹 F2E

SvgIcon 是全站統一的 icon 元件。它不內嵌 SVG 原始碼，而是輸出一個 <svg> 內含 <use href="…spritemap.svg#{icon}">，指向**建置時自動合併的 SVG sprite**。 所有 icon 原始檔放在專案根目錄的 _svg/，由 @spiriit/vite-plugin-svg-spritemap 合併為單一 spritemap.svg。

  • symbol id ＝ 檔名（去副檔名）：外掛設定 prefix: false，故 _svg/icon_house.svg → symbol id icon_house，icon prop 直接填這個名字。
  • 顏色：template 固定加 class="fill-current"，icon 以 currentColor 上色，因此顏色由外層 text-* 決定（例：text-[--green-8b0d]）。
  • 尺寸：元件本身不設寬高，由使用端以 Tailwind class 指定（例：class="h-[16px] w-[16px]"）。
  • 快取版號：sprite URL 帶 ?v=；prod 用建置 hash（spriteVersion = APP_HASH），dev 用時間戳，且訂閱 HMR 事件 project:svg-spritemap-update 在 icon 異動時即時刷新。

2. Props 說明 F2E

Prop型別必填預設說明
icon string 否 null sprite 內的 symbol id，等於 _svg/ 下的檔名（不含 .svg）。例：icon_house、chevron_left、icon_xmark。為 null / 不存在時 <use> 找不到對應 symbol，不會顯示任何圖形（不報錯、無 fallback）。

setClass 對應

本元件**無 setClass 機制**，也**無 :root CSS 變數**；外觀（尺寸 / 顏色）一律由使用端傳入的 Tailwind utility class 控制，故不提供 Playground。

3. 顏色與尺寸控制 UI

SvgIcon 使用 fill-current（即 fill: currentColor），顏色由外層的文字色繼承，尺寸由外層 h-* / w-* 控制：

<!-- 透過 text-color class 控制圖示顏色 -->
<CommonSvgIcon icon="icon_search" class="text-[--blue-08dc]" />

<!-- 透過 h-/w- class 控制尺寸 -->
<CommonSvgIcon icon="icon_search" class="h-[20px] w-[20px]" />
因為以 currentColor 上色，多色 / 固定色的 icon 不受 text-* 影響；只有單色 icon 能透過文字色換色。元件本身不設寬高，未指定 h-* / w-* 時會以瀏覽器預設尺寸顯示，建議一律明確指定。

4. 使用範例 F2E

基本用法

<CommonSvgIcon icon="icon_search" class="h-[20px] w-[20px] text-[--gray-64]" />

按鈕內的圖示

<button type="button" class="text-[--blue-08dc]">
  <CommonSvgIcon icon="icon_lang" class="h-[20px] w-[20px]" />
</button>

配合 mAnchor 的圖示

<BuyMAnchor
  text="下一頁"
  :config="{ icon: { name: 'chevron_right', position: 'right' } }"
  :setClass="{ icon: 'h-[12px] w-[12px]' }"
/>

動態 icon（綁定變數）

<CommonSvgIcon :icon="item.icon" class="h-[16px] w-[16px]" />
mAnchor 等共用元件不直接吃 icon 字串，而是透過自己的 config.icon（{ name, position }）轉呼叫 CommonSvgIcon，並用 setClass.icon 控制圖示尺寸。專案內實際用例：分頁箭頭（chevron_left / chevron_right）、清除鈕（icon_xmark）、收藏（icon_love_o）、Swiper 導覽鈕（:icon="navConfig.icon.prev"）等。

6. 狀態 / 例外行為 UI QA

  • icon 不存在 / 拼錯 / 為 null：<use> 參照不到 symbol，畫面不顯示任何圖形，不會報錯、也沒有 No Image fallback（與 ImgSrc 不同）。新增引用前請確認 _svg/ 內確有同名檔。
  • dev 環境：sprite 由 dev server 根路徑 middleware 提供（不在 _nuxt/ 下）；修改 / 新增 _svg/ 內檔案會觸發 HMR 事件 project:svg-spritemap-update，元件自動換新版號重抓，無需重整。
  • prod 環境：sprite 輸出為 _nuxt/{imgs}/svg/spritemap.svg，URL 帶建置 hash 版號（spriteVersion）作快取破壞。
  • 尺寸缺失：未給寬高 class 時，SVG 會以瀏覽器預設尺寸顯示，建議一律明確指定 h-* w-*。

7. 維護注意 F2E

  • 新增 icon：放入 _svg/（建議沿用 icon_* / chevron_* 命名）→ symbol id 即檔名，無需改元件。
  • sprite 來源 / 輸出路徑由 nuxt.config.ts 的 VitePluginSvgSpritemap 與 spritePath / spriteVersion（runtimeConfig.public）設定，調整路徑需同步檢查本元件的 spriteHref 計算邏輯。
  • icon 預設交由 SVGO（ViteImageOptimizer）最佳化，設定保留了 id（cleanupIds.remove: false），勿任意開啟移除 id，以免破壞 <use> 參照。
