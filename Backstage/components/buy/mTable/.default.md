# m-table --default

`components/buy/mTable/Default.vue` 是預設表格組件。後續若有不同版型的 table，會用 `--狀態` 的方式擴充。

## Props

| 名稱 | 型別 | 預設值 | 說明 |
| --- | --- | --- | --- |
| `thead` | `Array` | `null` | 表頭欄位設定。 |
| `tbody` | `Array` | `null` | 表格內容資料。可傳入陣列、空陣列或 `null`。 |
| `tfoot` | `Array` | `null` | 表尾資料。沒有資料時不顯示。 |
| `table` | `Object` | `null` | 完整表格資料。傳入後會優先使用 `table.thead`、`table.tbody`、`table.tfoot`。 |
| `config` | `Object` | `{}` | 組件設定，會與預設設定合併。 |
| `setClass` | `Object` | `{}` | 外部 class 設定，會與預設 class 合併。 |

## Data Source

可分開傳入 `thead`、`tbody`、`tfoot`：

```vue
<BuyMTableDefault :thead="thead" :tbody="tbody" :tfoot="tfoot" />
```

也可以傳入完整 `table`：

```vue
<BuyMTableDefault :table="table" />
```

當 `table` 有值時，組件會使用：

```js
thead = table.thead
tbody = table.tbody
tfoot = table.tfoot
```

## Thead

`thead` 或 `table.thead` 的欄位結構：

```js
const thead = [
  {
    id: 'name',
    label: '姓名',
    type: null,
    format: 'YYYY-MM-DD',
    colspan: {
      thead: null,
      tbody: null,
    },
    rowspan: {
      thead: null,
      tbody: null,
    },
    class: {
      thead: null,
      tbody: null,
      tfoot: null,
    },
  },
]
```

| 名稱 | 說明 |
| --- | --- |
| `id` | 欄位 key，同時也是 tbody slot name。 |
| `label` | 表頭文字，使用 `v-html` 輸出。 |
| `type` | 預設欄位輸出格式。可傳 `date`、`comma`、`currency`，沒有值時直接輸出原值。 |
| `format` | `type: 'date'` 時使用的日期格式，預設 `YYYY-MM-DD`。 |
| `colspan.thead` | 表頭 `th` 的 `colspan`。 |
| `colspan.tbody` | 內容 `td` 的 `colspan`。 |
| `rowspan.thead` | 表頭 `th` 的 `rowspan`。 |
| `rowspan.tbody` | 內容 `td` 的 `rowspan`。 |
| `class.thead` | 表頭 `th` 額外 class。 |
| `class.tbody` | 內容 `td` 額外 class。 |
| `class.tfoot` | 表尾 `td` 額外 class。 |

## Tbody

`tbody` 或 `table.tbody` 可傳入陣列、空陣列或 `null`。

```js
const tbody = [
  {
    name: '王小明',
    tel: '0912-345-678',
  },
]
```

欄位值會用 `thead.id` 對應資料物件：

```js
value = tbody[rowIndex][thead.id]
```

如果沒有提供對應的 named slot，組件會交給 `TBodyValue.vue` 依 `thead.type` 輸出該欄位值。

## Tfoot

`tfoot` 或 `table.tfoot` 沒有資料時不顯示。

表尾欄位會依照 `thead` 產生，並提供 `${thead.id}-tfoot` slot。

## Config

預設設定：

```js
const config = {
  isTheadFixed: false,
  noData: {
    icon: null,
    message: '尚無資料',
  },
}
```

| 名稱 | 說明 |
| --- | --- |
| `isTheadFixed` | 是否啟用固定表頭。啟用後會依 `.m-table-container` 的捲動位置移動同一個 `thead`。 |
| `noData.icon` | 無資料時傳給 `CommonSvgIcon` 的 icon。沒有值時不顯示 icon。 |
| `noData.message` | 無資料文字，使用 `v-html` 輸出。 |

### isTheadFixed

`isTheadFixed` 為 `true` 時，組件會監聽 `.m-table-container` 的 `scroll`，並用 `transform` 移動同一個 `thead`。`thead` 仍保留在 table flow 裡，不另外建立表頭結構，也不額外設定 `m-table-thead-th` 寬度。

## Slots

### tbody 欄位 slot

slot name 會使用 `thead.id`。

```vue
<BuyMTableDefault :thead="thead" :tbody="tbody">
  <template #name="{ item, value, index, column }">
    <p>{{ value }}</p>
  </template>
</BuyMTableDefault>
```

slot props：

| 名稱 | 說明 |
| --- | --- |
| `item` | tbody 當前列的完整資料。 |
| `value` | `item[thead.id]` 的欄位值。 |
| `index` | tbody 當前列 index。 |
| `column` | 當前 thead 欄位設定。 |

### tfoot 欄位 slot

slot name 會使用 `${thead.id}-tfoot`。

```vue
<template #name-tfoot="{ item, value, index, column }">
  <p>{{ value }}</p>
</template>
```

### footer slot

有傳入 `footer` slot 時，會在 table 後方顯示 `.m-table-footer`。

```vue
<template #footer>
  <BuyMPagination />
</template>
```

## Template Structure

```text
div.m-table.--default
  div.m-table-container
    table.m-table-content
      thead.m-table-thead
        tr.m-table-thead-tr
          th.m-table-thead-th
            p
      tbody.m-table-tbody
        tr.m-table-tbody-tr
          td.m-table-tbody-td
            slot
        tr.m-table-tbody-tr.--empty
          td.m-table-tbody-td.--empty
            div.m-table-no-data
              CommonSvgIcon
              p
      tfoot.m-table-tfoot
        tr.m-table-tfoot-tr
          td.m-table-tfoot-td
            slot
  div.m-table-footer
    slot name="footer"
```
